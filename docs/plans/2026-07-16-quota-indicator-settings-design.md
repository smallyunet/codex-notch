# 额度指示器与设置界面设计

## 目标

让刘海里的额度显示不再被单一的环形方向限制。用户可以在原生 macOS 设置窗口中即时切换两种样式，并在实际刘海里观察效果。

## 视觉方案

1. **顺时针圆环（默认）**：填充弧的末端固定在 12 点钟方向，额度下降时缺口从这里沿顺时针展开。它最接近常见进度环，也保留现有紧凑布局。
2. **波浪球**：圆形容器内用液面高度表示剩余额度，运行时波浪缓慢流动；完成时保留一次轻微脉冲。它在小尺寸下比环形更像一个状态徽标。波浪球可以选择把数字放在球内，或放到球的右侧；球旁模式不在液面上覆盖文字。

两种样式都复用同一套额度颜色：100% 为绿色，50% 为中间色，0% 为红色；运行中的动画仍使用当前额度对应的颜色。

## 设置与数据流

`QuotaDisplayStyle` 和 `QuotaLabelPlacement` 使用 `UserDefaults` 持久化。`NotchView` 和设置窗口都通过同一个 `@AppStorage` 键读取，因此设置修改后不需要重启即可更新刘海。设置窗口通过 SwiftUI `Settings` scene 提供，刘海右键菜单和应用菜单都提供入口。数字位置选项只在波浪球样式下显示，默认使用球内数字。

## 验证

- 为样式枚举、数字位置存储键和进度方向添加单元测试。
- 跑完整 Swift 测试套件并重新构建签名发布包。
- 重启应用，分别截图确认两种样式在紧凑刘海中不遮挡文字、不产生悬停闪烁；运行状态下左侧 ChatGPT 脉冲应清晰但不刺眼。
